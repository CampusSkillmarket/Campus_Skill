## 任务管理框架

### SMART 目标设定
```
S - Specific（具体的）
  目标要清晰明确，不要模糊

M - Measurable（可衡量的）
  有具体的衡量标准

A - Achievable（可实现的）
  在能力范围内，有挑战但可实现

R - Relevant（相关的）
  与长期目标相关

T - Time-bound（有时限的）
  有明确的截止日期

❌ 模糊目标：学好英语
✅ SMART 目标：3 个月内雅思达到 7 分，每天学习 2 小时
```

### 任务分解 WBS
```
项目：完成毕业论文

一级分解：
├── 1. 选题阶段
├── 2. 文献调研
├── 3. 研究设计
├── 4. 数据收集
├── 5. 数据分析
├── 6. 论文写作
└── 7. 修改定稿

二级分解（以"论文写作"为例）：
├── 6.1 引言（2 天）
├── 6.2 文献综述（3 天）
├── 6.3 研究方法（2 天）
├── 6.4 结果分析（5 天）
├── 6.5 讨论与结论（3 天）
└── 6.6 摘要参考文献（2 天）
```

### 优先级排序
```
艾森豪威尔矩阵

         紧急        不紧急
重要   │ 立刻做  │  计划做  │
       │ 危机    │  规划    │
       │ 紧急问题│  学习    │
───────┼────────┼─────────┤
不重要 │ 委托做  │  尽量不做│
       │ 会议    │  刷手机  │
       │ 琐事    │  八卦    │

使用原则：
1. 优先做"重要不紧急"（预防危机）
2. 快速处理"重要紧急"
3. 委托或批量处理"不重要紧急"
4. 尽量减少"不重要不紧急"
```

## 时间管理方法

### 番茄工作法
```
基本流程：
1. 选择一个任务
2. 设置 25 分钟倒计时
3. 专注工作，不做其他事
4. 时间到，休息 5 分钟
5. 每 4 个番茄后，休息 15-30 分钟

进阶技巧：
- 任务估算：预估需要几个番茄
- 番茄记录：记录实际使用的番茄数
- 中断处理：内部中断→记录后继续；外部中断→协商稍后

推荐工具：
- Forest（种树 APP）
- 番茄 TODO
- 潮汐（白噪音 + 番茄）
```

### 时间块规划
```
每日时间块模板

┌─────────────────────────────────────┐
│  06:00 - 07:00  │ 晨间例行          │
│  07:00 - 09:00  │ 深度工作块 1      │
│  09:00 - 10:00  │ 休息 + 运动       │
│  10:00 - 12:00  │ 深度工作块 2      │
│  12:00 - 14:00  │ 午餐 + 午休       │
│  14:00 - 16:00  │ 会议/协作块       │
│  16:00 - 18:00  │ 浅层工作块        │
│  18:00 - 20:00  │ 晚餐 + 放松       │
│  20:00 - 22:00  │ 学习/阅读         │
│  22:00 - 23:00  │ 睡前例行          │
└─────────────────────────────────────┘

规划原则：
- 黄金时间做最重要的事
- 批量处理同类任务
- 留出缓冲时间（20% 空白）
- 每天只安排 3 件大事
```

### GTD 方法
```
GTD 五步法

1. 收集（Capture）
   - 把所有事情记下来
   - 清空大脑
   - 工具：收件箱/笔记 APP

2. 处理（Clarify）
   - 这是可执行的任务吗？
   - 2 分钟内能完成吗？→ 立刻做
   - 需要多步骤？→ 拆解为项目
   - 委托他人？→ 加入等待清单

3. 组织（Organize）
   - 任务清单（Next Actions）
   - 项目清单（Projects）
   - 等待清单（Waiting For）
   -  someday/Maybe 清单

4. 执行（Engage）
   - 根据情境选择任务
   - 根据时间选择任务
   - 根据精力选择任务

5. 回顾（Reflect）
   - 每日回顾：检查今日完成
   - 每周回顾：清理收件箱，更新清单
```

```
我会提供：

【第一步】任务分解
每门课分解为：
- 看课件/笔记

- 背重点公式/概念
- 错题复习

【第二步】时间估算
| 科目 | 课件 | 真题 | 背诵 | 总计 |
|------|------|------|------|------|
| 高数 | 4h   | 6h   | 4h   | 14h  |
| 英语 | 2h   | 4h   | 6h   | 12h  |
| 专业 1 | 6h  | 4h   | 4h   | 14h  |
| 专业 2 | 6h  | 4h   | 4h   | 14h  |
| 政治 | 4h   | 4h   | 8h   | 16h  |
| 总计 | 22h  | 22h  | 26h  | 70h  |

【第三步】优先级排序
- 专业 1、专业 2：学分高，优先
- 高数：容易挂科，次优先
- 英语、政治：考前突击即可

【第四步】日程安排（7 天计划）
第 1-2 天：专业 1（14h）+ 专业 2（7h）
第 3-4 天：专业 2（7h）+ 高数（14h）
第 5 天：英语（12h）+ 政治（8h）
第 6 天：政治（8h）+ 机动
第 7 天：总复习 + 休息

【每日执行】
08:00-11:30 深度复习块 1（3.5h）
14:00-17:30 深度复习块 2（3.5h）
19:00-21:00 背诵块（2h）

【技巧】
- 番茄工作法：50 分钟学习 +10 分钟休息
- 交替复习：文理科交替，避免疲劳
- 费曼学习法：假装教别人
```

### 示例 2：创业项目时间管理
```
用户：我在创业，还要上课，时间不够用

我会提供：

⏰ 时间管理方案

【时间审计】
先记录一周时间花费，找出：
- 时间黑洞（刷手机、无效社交）
- 高效时段（你什么时候最专注）
- 可优化空间（哪些可以委托/砍掉）

【角色定位】
你同时是：
- 学生（上课、作业、考试）
- 创业者（产品、运营、融资）
- 普通人（休息、社交、健康）

【时间分配建议】
上课日：
- 上课时间：认真听讲，不摸鱼
- 课间时间：处理创业琐事（回复消息）
- 晚上：3-4 小时深度工作（产品/代码）

周末：
- 周六：全天创业（10h+）
- 周日：上午学习，下午休息

【任务管理】
创业任务：
- 产品迭代 → 深度工作块
- 用户沟通 → 批量处理（固定时段回复）
- 团队会议 → 固定时间（如周三晚）

学习任务：
- 作业 → 利用碎片时间
- 复习 → 考前集中突击

【授权与外包】
- 创业：哪些可以交给合伙人/兼职？
- 学习：哪些作业可以组队完成？
- 生活：外卖 vs 做饭，时间成本核算

【工具推荐】
- Notion：项目管理 + 知识库
- 飞书：团队协作
- 番茄 TODO：专注计时
- 滴答清单：任务管理
```

## 拖延症治疗

### 拖延原因分析
```
1. 任务太难 → 分解为小步骤
2. 任务太模糊 → 明确第一步做什么
3. 没有紧迫感 → 设置截止日期
4. 追求完美 → 先完成再完美
5. 精力不足 → 调整作息，运动
```

### 5 分钟起步法
```
告诉自己：只做 5 分钟

- 不想写论文？→ 只写 5 分钟
- 不想运动？→ 只运动 5 分钟
- 不想学习？→ 只学 5 分钟

原理：
- 降低启动门槛
- 一旦开始，通常会继续
- 5 分钟也比 0 分钟好
```

### 环境设计
```
增加阻力：
- 卸载娱乐 APP
- 把手机放另一个房间
- 退出游戏群

减少阻力：
- 提前准备好学习资料
- 前一晚把运动服放床头
- 书签只留学习网站
```

## 使用提示

1. **说明当前困扰**：拖延/混乱/没时间
2. **目标是什么**：考试/项目/日常提升
3. **可用时间**：每天能投入多少时间
4. **偏好工具**：纸质/电子/APP

## 注意事项

- 工具是手段，不是目的
- 不要过度优化，行动最重要
- 找到适合自己的方法，不要照搬
- 定期回顾调整
