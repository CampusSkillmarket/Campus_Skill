## 创意生成框架

### 框架 1：SCAMPER 技法
```
S - Substitute（替代）
  什么可以被替代？材料/过程/人/地点

C - Combine（组合）
  可以和什么组合？功能/服务/场景

A - Adapt（改造）
  可以模仿什么？其他行业/产品

M - Modify（修改）
  可以修改什么？大小/形状/颜色/属性

P - Put to other uses（他用）
  还有什么其他用途？新场景/新用户

E - Eliminate（消除）
  可以去掉什么？简化/精简

R - Reverse（反向）
  可以颠倒什么？顺序/角色/假设
```

### 框架 2：痛点挖掘矩阵
```
用户类型 × 场景 × 问题

用户类型：
- 大学生
- 职场新人
- 自由职业者
- 小微企业主
- 宝妈

场景：
- 学习/工作
- 生活/娱乐
- 社交/沟通
- 消费/购物
- 健康/运动

问题：
- 效率低
- 成本高
- 体验差
- 信息不对称
- 资源不足
```


## 创意评估工具

### RICE 评分模型
```
R - Reach（覆盖人数）
  1-10 分，预计影响多少人

I - Impact（影响力）
  1-10 分，影响程度多大

C - Confidence（信心）
  0-100%，有多大把握

E - Effort（投入）
  1-10 分，需要多少投入

RICE Score = (R × I × C) / E
```

### SWOT 分析
```
Strengths（优势）
- 内部优势
- 核心竞争力

Weaknesses（劣势）
- 内部劣势
- 需要改进的地方

Opportunities（机会）
- 外部机会
- 市场趋势

Threats（威胁）
- 外部威胁
- 竞争风险
```

## 创意激发技巧

### 技巧 1：随机词联想
```
随机选一个词，和你的问题强行关联

问题：如何改进水杯？
随机词：手机

联想：
- 水杯 + 手机 = 智能水杯（提醒喝水）
- 水杯 + APP = 记录饮水量
- 水杯 + 社交 = 喝水打卡 PK
```

### 技巧 2：极端假设
```
假设现有条件都不存在

问题：如何开餐厅？
极端假设：
- 没有店面 → 外卖厨房/餐车
- 没有厨师 → 预制菜/自动化
- 没有服务员 → 自助/机器人
```

### 技巧 3：反向思考
```
反过来想问题

问题：如何让用户多买东西？
反向：如何让用户少买东西？
洞察：精选、品质、长期价值

问题：如何让学生更努力？
反向：如何让学生更不努力？
洞察：减少干扰、即时反馈、降低门槛
```

## 使用提示

1. **说明背景**：你的情况/资源/限制
2. **明确目标**：创业/作业/内容创作
3. **偏好方向**：线上/线下/技术/非技术
4. **时间投入**：兼职/全职/项目周期

## 注意事项

- 创意需要验证，不要直接执行
- 小步快跑，快速迭代
- 关注用户反馈
- 不要追求完美，先上线
